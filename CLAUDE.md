## Code Style Skill

### 技术栈
- Tailwind CSS v4 + daisyUI v5，使用 `light` 主题
- 禁止手写 CSS，所有样式通过 Tailwind utility class 实现
- 禁止引入其他 UI 框架（Ant Design / Element / MUI 等）

### 视觉风格
- 现代简约白：大量留白，克制用色，最多 1 个强调色
- 阴影最多 `shadow-sm`，禁止重阴影 / 渐变 / 花哨动画
- 仅允许 `transition` 过渡效果

### 模块化架构
- 每个功能/职责独立一个文件，单文件不超过 300 行
- 入口文件只负责组装和启动，不写业务逻辑
- 禁止一个文件里定义多个组件/类/路由模块

#### 前端
- 组件拆分：页面级 -> 区块级 -> 原子级，每个组件独立文件
- 自定义 hooks/composables 每个独立文件
- API 按资源/模块分文件（如 `api/user.ts`、`api/order.ts`）
- 状态管理 store 按功能域拆分

#### 后端
- 按 routes / controllers / services / models / middleware 分层
- 每个路由模块独立文件，主路由文件只做挂载
- 数据库操作封装到 repository/dao 层

### 日志规范
- 必须使用结构化 logger：
  - JS/TS: pino 或 winston
  - Python: logging 模块
  - Go: slog 或 zap
- 禁止裸 `console.log` / `print` / `fmt.Println`
- 日志格式必须包含：时间戳、模块名、函数名、具体信息
- 函数入口：记录函数名 + 关键入参
- 函数出口：记录返回值或状态
- catch/error：记录完整错误信息 + 上下文变量
- 敏感数据（密码/token）用 `***` 脱敏

### 日志级别
- DEBUG: 变量值、内部状态
- INFO: 业务关键节点
- WARN: 非致命异常、降级、重试
- ERROR: 捕获异常、失败操作
