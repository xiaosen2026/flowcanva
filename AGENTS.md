# AGENTS.md — AI 代理规则

## 必须审查
- API 接口操作：非用户明确要求的扣款行为（如大量 API 调用、购买资源等）一律禁止
- 不要猜测或臆断系统状态（如'上游限流'等），需要日志/证据支撑

## 工作目录与 Git 安全
- **所有操作限制在当前项目目录内**，禁止跑到盘符根目录或其他项目目录
- Git 操作仅限当前仓库，不得污染外层 git 仓库或与父目录仓库合并
- 每轮任务结束前检查 Git 状态，如有修改应主动提交（只提交本轮修改的文件）
- 禁止 `git init` 在非项目目录执行

## 基本规则
- 请使用简体中文回复
- 遵循项目现有的代码风格和命名约定
- 修改文件前先阅读相关代码，理解上下文
- 保持代码简洁，避免不必要的复杂性
- 添加必要的注释说明关键逻辑

## 模块化架构
- 每个功能/职责独立一个文件，单文件不超过 300 行
- 入口文件只负责组装和启动，不写业务逻辑
- 禁止一个文件里定义多个组件/类/路由模块

## 日志规范
- 使用结构化 logger（JS/TS: pino/winston, Python: logging, Go: slog/zap）
- 禁止裸 console.log / print / fmt.Println
- 日志格式：时间戳、模块名、函数名、具体信息
- 敏感数据（密码/token）用 *** 脱敏
- DEBUG: 变量值/内部状态 | INFO: 业务关键节点 | WARN: 非致命异常 | ERROR: 捕获异常
